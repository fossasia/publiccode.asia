Public Money Public Code

Waarom is software die geschreven is met belastinggeld niet beschikbaar als Vrije Software?

Wij willen wetgeving die het nodig maakt dat publiek gefinancieerde software geschreven voor de publieke sector beschikbaar gemaakt wordt onder een Vrije en Open Bronlicentie. Als het publiek geld is, dan moet het ook publieke code zijn. In navolging van initiatieven wereldwijd vragen we om de release van code die wordt betaald door het publiek.

Code die betaald is door iedereen moet beschikbaar zijn voor iedereen!

Klinkt ingewikkeld? Nee, het is heel simpel!

Meer voordelen

Redenen voor Publieke Code

Besparen op belastingen

Soortgelijke toepassingen hoeven niet opnieuw van de grond af ontwikkeld te worden.

Samenwerking

Grote projecten kunnen samenwerken om expertise en kosten te delen.

Algemeen belang

Programma's betaald door het publiek zouden voor iedereen beschikbaar moeten zijn.

Innovatie bevorderen

Met transparante processen hoeft het wiel niet opnieuw uitgevonden te worden.

Vrije Software geeft iedereen het recht om software te gebruiken, te begrijpen, aan te passen en te delen. Deze rechten helpen het ondersteunen van andere fundamentele vrijheden zoals de vrijheid van meningsuiting, pers en privacy.

Gelooft u dat Vrije Software de standaardoptie zou moeten zijn voor publiek gefinancieerde software? Laten we onze politieke vertegenwoordigers overtuigen!

In onze open brief eisen we:

“Creëer wetgeving die vereist dat publiek gefinancierde software ontwikkeld voor de publieke sector openbaar beschikbaar gesteld moet worden onder een Vrije en Open Bron Softwarelicentie.”

1 organisaties en 51 individuen hebben onze oproep al gesteund door de Open Brief te ondertekenen. Help ons om een verschil te maken door het ook te ondertekenen! Wij zullen alle handtekeningen overhandigen aan vertegenwoordigers in heel Azië die debatteren over softwarevrijheid in overheden.
